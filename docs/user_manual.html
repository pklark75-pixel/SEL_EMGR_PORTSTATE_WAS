시스템 개요

01

PFS Console은 HSBC Korea 고객의 펀드 잔고 안내 이메일을 배치 처리하여 DB 발송 큐에 저장하고 관리하는 운영 화면입니다.

처리 흐름

배치 ZIP 업로드 ↓ CSV 파일 6건 파싱 (KRHSBC × 3 + KRHRED × 3) ↓ 고객별 HTML 이메일 생성 ↓ DB 발송 큐 INSERT (SMTP_INFO / SMTP_RECV / SMTP_FILES) ↓ 외부 발송 시스템이 큐를 읽어 발송
🔐

로그인 / 로그아웃

02

로그인

  1. 브라우저에서 http://<서버>:9080/pfs/ 접속
  2. 아이디 / 비밀번호 입력 후 로그인
항목기본값비고
아이디pfsadmin환경변수 PFS_APP_USER로 변경 가능
비밀번호pfs1234환경변수 PFS_APP_PASSWORD로 변경 가능

로그아웃

화면 우측 상단 [로그아웃] 버튼을 클릭합니다.

Batch ZIP Upload

05

필수 파일 구성

파일명 형식: (KRHSBC|KRHRED)_UT_YYYYMMDD_1_(CustAttr|CustSubs|EmailCustMast).txt

파일 수는 4개 또는 6개를 허용합니다. 필수 4개가 있으면 처리되고, CustSubs 2개는 선택입니다.

출처파일 종류파일명 예시필수
KRHSBC고객 속성KRHSBC_UT_20240115_1_CustAttr.txt✓ 필수
KRHSBC이메일 고객 마스터
수신자 이름·이메일 포함		KRHSBC_UT_20240115_1_EmailCustMast.txt✓ 필수
KRHSBC고객 구독
수신 동의 확인용 (현재 미사용)		KRHSBC_UT_20240115_1_CustSubs.txt선택
KRHRED고객 속성KRHRED_UT_20240115_1_CustAttr.txt✓ 필수
KRHRED이메일 고객 마스터
수신자 이름·이메일 포함		KRHRED_UT_20240115_1_EmailCustMast.txt✓ 필수
KRHRED고객 구독
수신 동의 확인용 (현재 미사용)		KRHRED_UT_20240115_1_CustSubs.txt선택

업로드 절차

  1. [파일 선택] 클릭 → .zip 파일 선택
  2. [Upload & Process] 버튼 클릭
  3. 처리 완료 후 상단에 결과 배너 확인
ℹ 업로드 완료 후 임시 배치 파일은 자동 삭제됩니다. HTML과 큐 데이터는 DB에만 보관됩니다.

성공 결과 메시지 해석

ZIP 업로드/큐저장 완료. imported=6, reportDate=2024-01-15,
queued(HSBC/HRED)=1250/430,
verify(files/recv/info)=1680/1680/1680,
smtpIdRange=1705123456000-1705123457000
항목의미
importedZIP에서 추출된 파일 수
reportDate배치 기준일
queued(HSBC/HRED)큐에 저장된 KRHSBC / KRHRED 고객 수
verify(files/recv/info)각 테이블 INSERT 확인 건수
smtpIdRange생성된 SMTP_ID 범위

제약 사항

항목제한
최대 파일 크기20MB (운영자가 변경 가능)
파일 형식.zip 만 허용
허용 파일 수4개 또는 6개 (필수 4개 미충족 시 검증 오류)
파일명 형식(KRHSBC|KRHRED)_UT_YYYYMMDD_1_종류.txt
모든 파일 날짜동일한 기준일(YYYYMMDD) 이어야 함

HTML 이메일 미리보기

08

Queue 테이블에서 [미리보기] 버튼을 클릭하면 해당 고객에게 발송될 HTML 이메일 내용을 모달 팝업으로 확인할 수 있습니다.

동작방법
미리보기 열기Queue 테이블 행의 [미리보기] 버튼 클릭
미리보기 닫기우측 상단 [✕] 클릭 / 배경 클릭 / ESC 키
ℹ HTML 내용은 DB의 EMGR_SMTP_FILES_Q 테이블에서 직접 조회됩니다.
🗑

큐 레코드 삭제

09
⚠ 삭제는 되돌릴 수 없습니다. 발송 대기 중인 레코드를 삭제하면 해당 고객에게 이메일이 발송되지 않습니다.

삭제 절차

  1. 삭제할 행의 체크박스 선택 (헤더 체크박스로 전체 선택 가능)
  2. 상단에 노란색 삭제 바 표시 → 선택 건수 확인
  3. [선택 삭제] 버튼 클릭
  4. 확인 대화상자에서 [확인] 클릭
ℹ 삭제 시 SMTP_INFO_Q, SMTP_RECV_LIST_Q, SMTP_FILES_Q 3개 테이블에서 동시에 삭제됩니다.
⟳

자동 새로고침

10

Queue 섹션 우측 상단의 [⟳ 자동 새로고침] 버튼을 클릭하면 30초마다 페이지가 자동 갱신됩니다.

상태버튼 표시
비활성⟳ 자동 새로고침
활성 (파란색)⟳ ON (30s)

발송 시스템이 큐를 처리하는 동안 발송 상태 변화를 실시간으로 모니터링할 때 유용합니다.
