환경 설정

04

pfs-jdk.properties (운영 설정)

파일 위치: <pfs.baseDir>/pfs-jdk.properties

# ── DB 연결 (필수) ──────────────────────────────
db.queue.enabled=true
db.url=jdbc:postgresql://<호스트>:5432/<DB명>
db.user=<DB사용자>

# ── DB 비밀번호 (3가지 방식 중 택일) ───────────
# 방식 1: 평문 (비권장, 개발용)
db.password=<비밀번호>

# 방식 2: AES 암호화 + 환경변수 키
db.password.enc=<암호화된비밀번호>
db.password.keyEnv=PFS_DB_KEY

# 방식 3: AES 암호화 + 키 파일
db.password.enc=<암호화된비밀번호>
db.password.keyFile=C:\keys\pfs.key

# 방식 4: AES 암호화 + Windows Credential Manager
db.password.enc=<암호화된비밀번호>
db.password.keyWinCred=PFS_DB_CRED

# ── 메일 발신 정보 ──────────────────────────────
mail.appName=eManager
mail.sender=HSBC Korea <info@kr.hsbc.com>
mail.title=HSBC 은행 펀드 잔고 안내

# ── 운영 파라미터 (선택) ────────────────────────
pfs.upload.zipMaxBytes=20971520       # 최대 ZIP 크기 (기본 20MB)
pfs.upload.historyRows=12             # 이력 표시 건수
pfs.queue.recentRows=20               # 큐 기본 조회 건수

# ── 테이블명 (기본값, 변경 시만 지정) ──────────
db.queue.filesTable=LOAN.EMGR_SMTP_FILES_Q
db.queue.recvTable=LOAN.EMGR_SMTP_RECV_LIST_Q
db.queue.infoTable=LOAN.EMGR_SMTP_INFO_Q
db.audit.uploadTable=LOAN.EMGR_UPLOAD_AUDIT

DB 비밀번호 암호화

운영 환경에서는 평문 비밀번호 대신 AES 암호화를 권장합니다.

  1. 암호화 도구 실행: java -cp portstate-was.war com.hsbc.sel.emgr.jdk.PfsPasswordCryptoTool
  2. 암호화 키와 DB 비밀번호 입력 → 암호화된 문자열 출력
  3. 출력된 값을 db.password.enc에 저장
  4. 키는 환경변수(PFS_DB_KEY), 키 파일, 또는 Windows Credential Manager 중 하나에 저장
ℹ 키 우선순위: 환경변수 → 키 파일 → Windows Credential Manager 순으로 조회합니다.

로그인 계정 변경

운영 환경에서는 환경변수로 계정 정보를 관리하세요.

# Windows 환경변수 또는 WLP jvm.options
PFS_APP_USER=pfsadmin

05

최초 배포 시 PostgreSQL에 아래 테이블을 생성해야 합니다.

-- 발송 정보 테이블
CREATE TABLE LOAN.EMGR_SMTP_INFO_Q (
    SMTP_ID     NUMERIC(20)  NOT NULL,
    APP_NAME    VARCHAR(20),
    REF_NO      VARCHAR(50),
    SENDER      VARCHAR(255),
    TITLE       VARCHAR(255),
    CONTENT     VARCHAR(3000),
    C_TIME      TIMESTAMP,
    S_CNT       INTEGER      DEFAULT 0,
    S_FLAG      VARCHAR(1)   DEFAULT 'N',
    SOURCE_APP  VARCHAR(10),
    FUND_COUNT  INTEGER,

);

-- 수신자 목록 테이블
CREATE TABLE LOAN.EMGR_SMTP_RECV_LIST_Q (
    SMTP_ID       NUMERIC(20)  NOT NULL,
    APP_NAME      VARCHAR(20),
    RECEIVER      VARCHAR(255),
    RECEIVER_NAME VARCHAR(100),
    PRIMARY KEY (SMTP_ID, APP_NAME)
);

-- HTML 파일 테이블
CREATE TABLE LOAN.EMGR_SMTP_FILES_Q (
    SMTP_ID  NUMERIC(20)  NOT NULL,
    APP_NAME VARCHAR(20),
    NAME     VARCHAR(255),
    DATAFILE BYTEA,
    PRIMARY KEY (SMTP_ID, APP_NAME)
);

-- 업로드 감사 로그 테이블
CREATE TABLE LOAN.EMGR_UPLOAD_AUDIT (

    HSBC_QUEUED    INTEGER,
    HRED_QUEUED    INTEGER,
    SUCCESS_FLAG   VARCHAR(1),
    MESSAGE        VARCHAR(1000),
    UPLOAD_USER    VARCHAR(100)
);
⚠ 테이블명은 pfs-jdk.properties에서 변경 가능합니다. 변경 시 DDL과 설정 파일을 동시에 수정하세요.

WLP 배포

06

로컬 기동 (개발/테스트)

# 빌드 + WLP 기동 (포트 9080)
mvn liberty:run

# 접속 확인
http://localhost:9080/pfs/

운영 서버 배포

  1. WAR 빌드: mvn clean package -DskipTests
  2. WAR 파일 복사: target/portstate-was-1.0.0-SNAPSHOT.war → %WLP_HOME%\usr\servers\pfsServer\apps\
  3. pfs-jdk.properties를 <pfs.baseDir>/ 에 배치
  4. 서버 기동: %WLP_HOME%\bin\server.bat start pfsServer

server.xml 핵심 설정

<server description="PFS Portfolio Statement WLP Server">

    <featureManager>
        <feature>servlet-6.0</feature>
    </featureManager>

    <httpEndpoint id="defaultHttpEndpoint"
                  host="*"
                  httpPort="9080"
                  httpsPort="9443"/>

    <webApplication id="portstate-was"
                    location="portstate-was-1.0.0-SNAPSHOT.war"
                    contextRoot="/pfs">
        <!-- Spring Boot 클래스가 WLP 내장 클래스보다 우선 로드 -->
        <classloader delegation="parentLast"/>
    </webApplication>

</server>
⚠ classloader delegation="parentLast" 설정은 필수입니다. 이 설정이 없으면 Spring Boot 클래스와 WLP 내장 클래스 충돌이 발생합니다.

JVM 옵션 설정 (jvm.options)

# %WLP_HOME%\usr\servers\pfsServer\jvm.options
-Dpfs.baseDir=C:\pfs\files
-Xmx512m
-Xms256m

  1. 새 WAR 파일 빌드
  2. apps/ 폴더의 기존 WAR 파일을 새 파일로 교체

오류 대응

10

화면 오류 배너 유형

오류 메시지원인조치
DB 연결 실패PostgreSQL 연결 불가db.url, db.user, 비밀번호 및 DB 서버 상태 확인
DB queue is disableddb.queue.enabled=falsepfs-jdk.properties에서 true로 변경
ZIP 업로드 후 검증 실패필수 파일 미충족 또는 파일명 형식 오류파일명 형식 확인: (KRHSBC|KRHRED)_UT_YYYYMMDD_1_종류.txt
필수 4개: CustAttr + EmailCustMast × 2출처
ZIP 파일 크기 초과20MB 초과ZIP 최적화 또는 pfs.upload.zipMaxBytes 증가
DB driver loading failedPostgreSQL JDBC 드라이버 누락WAR 내 postgresql-42.7.3.jar 포함 확인
No key source configured암호화 키 미설정환경변수 PFS_DB_KEY 또는 db.password.keyFile 설정

서버가 기동되지 않을 때

  1. logs\messages.log에서 CWWKZ0009E 또는 CWWKE0002E 메시지 확인
  2. logs\ffdc\ 폴더에서 상세 스택 트레이스 확인
  3. server.xml의 classloader delegation="parentLast" 설정 확인
  4. pfs-jdk.properties 파일 경로 및 문법 오류 확인

DB 연결 재시도 동작

읽기 전용 DB 조회 메서드에 Spring Retry가 적용되어 있습니다.

설정값
최대 재시도 횟수3회
초기 대기 시간1초
백오프 배수2배 (1초 → 2초 → 4초)
적용 대상큐 조회, 건수 조회, 통계, 업로드 이력 조회
ℹ 쓰기 작업(ZIP 업로드, 큐 삭제)은 중복 데이터 방지를 위해 재시도가 적용되지 않습니다.
